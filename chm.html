Circular Heatmap

A novel visualization for comparing cyclic data

  • Visualization
  • Front-end Development
  • Hackathon Submission

This is a Hackathon challenge conducted by MicroStrategy. Over 200 global employees and 61 teams participated, I was placed second and recieved special shout-out from the CTO.

Goal Design and build a visualization that will make MicroStrategy better.

My role Designer and developer

Challenges
  • MicroStrategy Visual Insight already supports a number of basic visualization types, so that it is hard to find an existing visualization to improve the already large visualization gallery.
  • It is very time-consuming to build a customized visualization. The time allowed for this challenge is very limited (less than 3 days).
  • The final product must be integrated into MicroStrategy code base and work seamlessly.
  • The final product needs to be very visually stunning to stand out in the competition.

“It is beautiful, elegant, and very polished. Feels like we can ship it to the customers tomorrow.”

- Peng Xiao, CTO. During the closing speech at the Hackathon finale

Screenshot

This is the final visualization. Build with Mojo (MicroStrategy native JavaScript framework) and D3.js.

Details

I decided to create a heatmap which is a great addition to the current visualizations. The current MicroStrategy VI only supports treemap. Treemap is excellent at showing all the data when there is a lot of categories, but it is not quite suffcient for comparison. On the other hand, a regular heatmap preserves the position for each item so it is very easy to compare.

I chose to make it circular so that it is 1) visually interesting, 2) it gives viewer a sense of repetition, so it is especially good for finance and sales data (such as monthly revenue).

I also chose to present it on a black background so that any color in the visualization will pop up.

By making the visualization circular, I inevitably made it hard to compare data than a regular heatmap. To solve that, I created a hover effect on both radial and segmentational direction. User can mouse over any radial or segmentational label to highlight the color in the selection, while all other unit is faded out.

To view the actual metric values, I created a center display that fit perfectly in the hollow area so no additional screen space is needed. I've also created a "fill" transitional animation so that user has a secondary visual cue about how exactly the values compares to each other.

Take away

Attention to details always win. My strategy for this challenge is to produce something polished. I've tested the visualizations on different datasets. I also created transitional animation for any data changes. I took a lot of time just to find out the perfect padding and font-size. And apparently these efforts has paid off.

Practice makes perfect. One of the reason for my success is my presentation. This is the step where most of the contestants overlook. I not only spend time to build this visualization, but I also spent a lot of time on the presentation slides and practiced the presentation multiple times until I have the perfect timing. I also tried to make the presentation fun and engaging. This made a big difference for me.
